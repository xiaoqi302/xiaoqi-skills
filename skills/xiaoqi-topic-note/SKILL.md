# 小七主题笔记图

把一个主题转成一张竖版中文手绘笔记图。保留用户主题作为标题锚点，不文学化改写，不扩成教程、长文、思维导图或多页内容。

详细检查与修复规则见 [references/qa-checklist.md](references/qa-checklist.md)。

## 工作流

1. 把用户原始输入确定为 `{主题}`，删除其中纯操作性措辞，但不改变主题含义。
2. 规划最多 3-5 个短模块：
   - 优先使用用户提供的关键词和事实。
   - 可以补充少量辅助标签，但只能使用已给来源、稳定常识或经过可靠来源核验的事实。
   - 涉及近期、冷门、争议或高风险信息时，先用可用检索工具核验；无法核验就不添加事实性断言。
   - 不为凑满模块而虚构内容；信息不足时允许只保留 1-2 个模块。辅助标签不是改标题，也不是扩写成长文。
3. 默认生成 `1` 张、竖版 `3:4`；只有用户明确要求时才改变比例或生成多张。本 Skill 不处理需要连续翻页叙事的多页轮播。
4. 区分交付模式：
   - 用户要求“生成 / 做图 / 出图”且环境有图像工具：实际生成图片。
   - 用户明确只要提示词，或环境没有图像工具：输出完整提示词并标注“仅提示词”；不要声称已经生成图片。
5. 实际生成时调用当前环境可用的图像工具。生成后查看真实成图，检查主题、中文、文字密度、模块数和留白。
6. 未通过时先局部编辑，否则重生成；每张图最多修复 `2` 轮。两轮后仍有问题，交付最优版本并指出剩余问题。
7. 安全落盘：优先采用用户给出的绝对输出目录；相对目录必须先基于绝对工作区根解析，并确认结果仍在工作区内；未指定目录时保存到 `<绝对工作区根>/assets/xiaoqi-topic-note/`。使用不覆盖的文件名；无法确定工作区时保留工具返回的原始绝对路径。

## 提示词骨架

```text
请生成一张小七风格中文手绘主题笔记图，主题标题严格使用：【{主题}】。

画面要求：
- 竖版 3:4，适合手机阅读，一张完整图片，不拼贴多页。
- 大标题明显偏大；总文字量低，只使用下面给出的短标签。
- 模块不超过 3-5 个；如果只提供 1-2 个标签，就保持更大留白，不自行补写。
- 辅助标签：{已核验辅助标签；没有则省略}。
- 黑色手写笔记线条，少量红色、蓝色或黄色马克笔高亮。
- 插图比文字更重要；可用箭头、波浪线、圈注、便签或轻微剪报感元素。
- 中文必须清晰准确；禁止密集小字、长段落、未经提供的事实、PPT 风和正式海报风。

输出一张高分辨率完整图片，不要英文主文案。
```

## 交付

- 实际生成：返回主题、绝对文件路径、检查结果与仍存在的问题。
- 仅提示词：直接给完整提示词，不虚构文件路径。

# 来源与证据规则

## 来源获取

按证据距离优先选择来源：

1. 一手来源：官方文档、原始公告、论文或预印本、数据集、监管或司法文件、公司披露、作者原始发言、原始数据。
2. 高质量二手来源：有署名、日期、引用链和编辑责任的专业媒体或研究机构。
3. 其他材料：聚合页、转载、社交帖子、用户粘贴内容，只能按其可验证程度使用。

用户材料也是来源，应分配 Source ID；没有原始出处时标记“用户提供，原始来源未提供”，不能默认视为已核事实。

搜索结果标题、摘要、AI 摘要和网页片段只用于发现来源，不能作为证据。必须打开实际页面或文件，确认正文支持主张后才能登记。

## Source ID 与来源登记

按 S001、S002 顺序编号。每份来源至少记录：

- 标题、作者或发布主体。
- 来源类型及一手/二手属性。
- 实际 URL 或本地文件名。
- 发布日期、事件日期、访问日期。
- 实际读取位置：网页标题/章节，或 PDF 的真实页码、章节、图表。
- 可支持的范围、限制和访问状态。

同一内容的转载、镜像或互相引用不算独立证据。记录其引用链，避免用多个副本制造“多方证实”。

## 三种日期

- 发布日期：来源对外发布或更新的日期。
- 事件日期：来源描述的事件实际发生日期。
- 访问日期：本次打开页面或文件的日期。

三者分列记录。未知时写“未提供/未确认”，不得从搜索排序、URL、文件名或模糊时间词中猜测。

当前价格、版本、职位、政策、产品能力、榜单、进度等易变信息必须在本次任务中重新查证，并记录访问日期。历史快照不能表述为当前状态。

## 主张分类

- 事实：来源直接支持、口径明确，且没有被可靠来源否定。
- 来源方主张：某个主体自行声称的能力、数字、计划或解释；即使来自官方也不自动升级为事实。
- 分析：基于已登记证据做出的解释、比较或推断，必须说明依据与边界。
- 待确认：缺来源、口径不清、相互冲突或尚未验证的内容。

“论文报告提升 40%”可作为论文报告行为的事实；“方法实际提升 40%”仍是需要核对测试口径的来源方主张。

## Claim ID 与证据绑定

按 C001、C002 顺序编号。每个关键 claim 必须：

- 标明分类。
- 绑定至少一个实际读取过的 Source ID。
- 给出可定位的页码、章节、表格或网页位置；没有可靠定位时写“未提供”，不编页码。
- 标记状态：已支持、单一来源、存在冲突或待确认。
- 分析类 claim 绑定其事实依据，不得只引用另一段分析。

没有 Source ID 的想法只能放入“问题/缺口”，不能进入 claim ledger 的关键结论。

## 冲突处理

来源冲突时保留双方，不擅自平均或选择更顺眼的数字。比较其发布时间、原始性、口径、版本和利益关系，并写明需要什么证据才能解决。

无法打开正文、只看到摘要、PDF 缺页或网页已更新时，明确记录访问限制；不要据此补造链接、原文、页码、日期或结论。

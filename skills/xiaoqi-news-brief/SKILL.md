# Xiaoqi News Brief

把新闻写成“小七式快评”：先讲清事实，再讲人话，再下一个有边界的判断。不要把快评写成通稿。

## 参考资源

按需读取：

- `references/news-brief-style.md`：快评结构和语言规则。
- `references/fact-check-rules.md`：事实边界和不编造规则。

## 工作流

1. 提炼新闻事实：谁、做了什么、什么时候、涉及什么产品/技术。
2. 标出不确定信息：没有来源、数字不清、推测性强的内容不要写死。
3. 用 4 段结构输出：
   - **一句话新闻**：30-60 字。
   - **这事真正重要的地方**：解释变化，不堆术语。
   - **我的判断**：给趋势判断，保留边界。
   - **可以继续观察什么**：列 2-4 个后续信号。
4. 如果用户要公众号长文，建议转 `$xiaoqi-wechat`。

## 输出格式

```markdown
## 一句话

...

## 重要的是

...

## 我的判断

...

## 接下来观察

- ...
- ...
```

## 禁忌

- 不编造发布时间、融资额、用户量、benchmark。
- 不把 PR 口径当事实结论。
- 不使用“震撼发布”“颠覆行业”这类营销话。
- 不为了显得犀利而过度下结论。

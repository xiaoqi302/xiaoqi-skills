# 新闻事实边界

## 来源记录

优先使用一手来源：官方公告、产品文档、论文、监管文件、公司披露或当事人原始发言。媒体报道可补充背景，但不得把转述升级为一手证据。

至少记录：

- 原始来源或“用户提供材料，原始来源未提供”。
- 来源发布日期。
- 事件实际发生时间；若与发布日期不同，分别写明。
- 核心数字、引语和结论分别来自哪里。

没有日期时写“材料未提供日期”，或删除不必要的时间表达。禁止用“近日”“最近”“刚刚”掩盖日期缺失。

## 主张分类

- 已核事实：来源直接支持且口径明确。
- 来源方主张：公司、项目方或当事人自行宣称，使用归属表达。
- 作者判断：基于事实的解释或趋势判断，说明依据和限制。
- 待确认信息：来源不明、相互矛盾或无法复核，不进入确定性正文。

事实句不夹带评价；判断段不新增事实。推测必须明确使用“可能”“如果条件成立”等边界词，并给出可验证信号。

## 核查重点

- 产品能力是已经上线、灰度测试、宣布计划，还是仅做演示。
- benchmark 是否说明数据集、版本、评价口径和对照基线。
- 融资、营收、用户量和商业合作是否有可追溯来源。
- 新闻发布时间是否被误当成事件发生时间。
- PR 稿中的“首个、领先、颠覆”是否只是发布方口径。

当问题依赖当前状态时先查证；无法查证则明确“以下仅基于用户提供材料”，不暗示信息是最新的。

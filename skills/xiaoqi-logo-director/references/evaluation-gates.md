## Gate 1: Big-Tech Logo Fitness

Passes if:
- simple enough to memorize
- one clear visual hook
- scalable to 32px
- can become app icon, favicon, loading mark, and button icon
- no unnecessary texture, shadow, or decorative complexity

Fails if:
- looks like an illustration
- depends on background effects
- has too many elements
- needs explanation to understand the hook

## Gate 2: Brand Collision

Reject if it resembles:
- Claude-style radial burst
- OpenAI-style interwoven knot
- Gemini-style four-point sparkle
- Meta infinity
- Microsoft 2x2 window
- Google segmented G
- Figma vertical block stack
- Cursor arrow
- Perplexity angular line mark

Near-category resemblance is also a problem. Do not rescue a collision by changing color.

## Gate 3: Semantic Collision

Reject if first read is:
- folder
- browser tab
- app launcher grid
- component board
- workflow chart
- process diagram
- play button
- lightning bolt
- chart
- code bracket
- robot
- chat bubble
- random letter

These can be used as sources only after abstraction. The final mark should not be read literally.

## Gate 3.5: Logo vs UI Diagram

Reject if:
- it contains several rails or paths that read as workflow UI
- it looks like an icon explaining a feature instead of a brand mark
- it is a set of related symbols rather than one symbol
- it relies on multiple separated blocks without one memorable outer silhouette

Repair by reducing it to one compact emblem, one cutout mass, or one solid glyph.

## Gate 4: Color Discipline

Passes if:
- color improves recognition
- palette matches product maturity
- color family differs from other concepts in the same round
- black/white version would still work

Fails if:
- color hides weak structure
- all concepts are dark
- all concepts use similar blue/cyan gradient
- multi-color looks like Google, Microsoft, or Figma

## Gate 5: Original Motif

Passes if:
- the motif maps to the product's job
- the shape would feel odd on an unrelated product
- the hook is not generic AI decoration

Fails if:
- “AI” is the only explanation
- it could be used by any SaaS company
- the shape is a style trend, not a brand idea

## Gate 6: Actual Artifact Checks

After a mother mark is selected, create and visually inspect all of these from the same source asset:

- `32px` color preview
- monochrome preview
- reverse preview on a dark background
- `16px` and `32px` favicon previews
- App icon preview with the core mark unchanged

Reject the production adaptation if thin gaps disappear, the silhouette collapses, foreground and background merge, or any geometry differs from the selected master.

## Scoring

Score Fitness, Brand Collision, Semantic Collision, Color Discipline, and Original Motif separately from 1-5:

- 1: collision or unusable
- 2: polished but generic
- 3: acceptable icon, weak brand
- 4: strong candidate, worth iterating
- 5: ownable, scalable, distinctive

Any collision-gate failure is an automatic rejection. Otherwise, only keep concepts whose average is at least 4 and whose actual artifact checks pass.

## Clearance Boundary

These gates identify obvious visual and semantic collisions; they do not establish trademark availability, legal originality, or registrability. Commercial use requires separate jurisdiction-specific trademark and similar-device searches.

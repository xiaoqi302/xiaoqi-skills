# Logo Aesthetic Doctrine

Use the named market examples below only as category references. No local reference-logo asset set is assumed, and their symbols must never be copied.

## What The Market Examples Show

The market is diverse:
- **Single-color structural**: OpenAI, Anthropic, Cursor, Perplexity.
- **Warm humane symbol**: Claude, Hugging Face.
- **Bright system color**: Google, Microsoft, Gemini, Figma, Copilot.
- **Developer / infrastructure geometry**: Cursor, Perplexity, DeepSeek, LobeHub, Qwen.
- **Friendly product mascot / face**: Hugging Face, LobeHub.

The lesson is not “AI logos are dark and abstract.” The lesson is that successful logos own one memorable system.

## Shared Success Traits

Strong logos usually have:
- A clear silhouette before color.
- One dominant visual hook.
- A simple geometry system: repeated radius, repeated angle, repeated module, or repeated stroke width.
- Enough uniqueness to be redrawn from memory.
- Good small-size survival at 32px.
- A color strategy that strengthens the structure.
- A single brand idea, not a pile of AI cliches.

## Valid Style Families

Use these families to diversify each round.

### Structural Mono

Best for foundation models, infra, research labs.

Traits:
- black, ink navy, charcoal, or one dark color
- high negative-space discipline
- no gradients
- shape must carry all identity

Risks:
- can look like OpenAI / Anthropic / generic crypto
- can become too serious for apps

### Warm Humane

Best for assistants, learning tools, model brands with a human tone.

Traits:
- clay, terracotta, coral, ochre, warm orange
- organic geometry with controlled irregularity
- friendly but not childish

Risks:
- can become Claude-like starburst
- can become blob / flower

### Bright System

Best for apps, creative tools, productivity, consumer AI.

Traits:
- clear blue, green, yellow, coral, violet
- flat colors or restrained gradient
- modular but not standard app grid

Risks:
- can copy Google, Microsoft, Figma, Gemini
- can become decorative instead of structural

### Developer Geometry

Best for coding tools, agent workspaces, model routing, SaaS infra.

Traits:
- cobalt, cyan, mint, navy, or monochrome
- precise angles, cuts, brackets, slots, routing paths
- strong icon behavior

Risks:
- can become folder, arrow, portal, cube, or letter

### Friendly Character

Best for open source, community, education, onboarding.

Traits:
- simple face or soft character cue
- high memorability
- bright color

Risks:
- not suitable for serious infra
- easy to become mascot illustration instead of logo

## Anti-Pattern Library

Reject these unless the user explicitly asks:
- four-point sparkle
- six-loop knot
- infinity loop
- standard 2x2 color grid
- generic app launcher grid
- folder tab
- browser window
- code brackets as direct symbol
- play button
- lightning bolt
- central hole plus rounded panels
- S-shaped wave as main identity
- gradient ribbon without a strong silhouette

## Diversity Rule

Each 3-logo round should include:
- one shape-led logo
- one color-led but structure-safe logo
- one interface/product-led logo

Do not generate three variants of the same family.

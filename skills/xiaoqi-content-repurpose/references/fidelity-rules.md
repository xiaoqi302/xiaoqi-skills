# 内容忠实度规则

## 优先级

发生冲突时按以下顺序处理：

1. 原稿中的事实、引用、立场和确定性程度。
2. 用户明确指定的平台规格与表达要求。
3. 平台常见写法。

平台惯例不能覆盖原稿事实，用户的风格要求也不能授权虚构内容。

## 忠实度锁

改写前在内部建立六类锁定项：

| 类别 | 必须记录 | 允许变化 |
|---|---|---|
| 事实 | 人物、事件、对象、关系 | 句式和出现顺序 |
| 数字 | 数值、单位、日期、比例 | 数字周围的解释句式 |
| 引用 | 原话、说话人、出处 | 仅可缩短非关键上下文，不改引文本身 |
| 立场 | 核心判断、反对项、保留意见 | 论据顺序和段落结构 |
| 语气 | 人称、克制度、确定性 | 平台节奏和句长 |
| CTA | 行动目标、链接、期限 | 平台化措辞 |

原稿未提供的项目标记为“不可新增”，不能用常识补齐。

## 可以做

- 调整开头，使原稿中已有的核心判断更早出现。
- 重排论据，让平台读者更容易跟上。
- 删除重复，拆分长句，合并相邻解释。
- 把同一观点改成帖子、列表、邮件或口播结构。
- 在不改变含义的前提下使用平台自然的连接词和节奏。

## 不可以做

- 新增数据、案例、用户反馈、个人体验或业内共识。
- 把相关关系改成因果关系。
- 把推测、可能、倾向写成已经证明或必然发生。
- 改写引号内原话，或为引用虚构来源。
- 添加原稿没有的收益承诺、稀缺性、紧迫感或恐惧诉求。
- 伪造链接、账号、产品功能、优惠或截止日期。
- 为了“更像平台”改变作者立场和身份。
- 在原稿和用户要求都没有时，自行添加 Emoji、话题标签、网络口头禅或互动诱导。

## 交付前核对

逐个平台回答：

1. 每个事实、数字和引用能否在原稿中定位？
2. 核心立场和确定性是否与原稿一致？
3. 开头是否来自原稿，而不是新造的标题党承诺？
4. CTA 的行动目标、链接和期限是否保持？
5. 是否出现原稿没有的第一手体验或社会证明？
6. 该版本是否完整独立，而非原稿截断或其他版本的附注？

任一项不通过，先修正再交付。原稿本身存在冲突时保留原文并简短标注，不擅自替作者裁决。

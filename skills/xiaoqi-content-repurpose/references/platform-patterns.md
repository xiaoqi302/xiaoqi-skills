# 平台内容模式

这些是结构模式，不是固定长度规范。用户规格优先；需要精确字符数、媒体数量或发布限制时，查询平台官方当前规则。

## 小红书

- 标题：使用原稿已有的判断、问题或反差，不承诺原稿没有的结果。
- 正文：场景或问题 → 核心判断 → 2-5 个要点 → 原稿 CTA。
- 使用短段落和清晰层次；Emoji 与话题标签只在用户明确要求时添加，标签只能来自原稿主题。
- 不伪造亲测体验、消费经历或“所有人都在做”的社会证明。

## X / Threads

- 单帖：核心判断 → 最必要依据 → 结论或 CTA。
- 串文：首帖给完整主张；后续每帖推进一个论点；末帖收束，不重复首帖。
- 是否拆串由内容完整性和用户规格决定，不预设固定字符数。

## 即刻

- 用一条明确观察或判断开场。
- 依次补背景、关键依据和作者结论，保持口语但不假装随手亲历。
- 结尾可以留问题或 CTA，但必须符合原稿目的。

## 朋友圈

- 保留一个核心判断，用自然短段落说明缘由。
- 弱化栏目感和格式感，不强塞小标题、标签或营销话术。
- 原稿没有销售或互动 CTA 时，不额外添加。

## LinkedIn

- 专业判断开场 → 背景 → 关键洞察 → 对工作或行业的含义 → CTA。
- 保留证据与限定条件，不虚构业务结果、客户案例或职位经历。
- 可调整为清晰段落或要点，但不能把观点包装成未经证明的方法论。

## 邮件摘要

- 输出主题建议和完整邮件正文。
- 正文使用开场说明 → 核心摘要 → 关键要点 → CTA。
- 只有原稿提供链接、附件或截止日期时才写入；不要生成假链接占位。

## 短视频口播

- 口播开场 → 背景铺垫 → 2-4 个内容节拍 → 核心结论 → 原稿 CTA。
- 使用可直接朗读的短句、停顿和自然转折，删除书面重复。
- 不自动生成分镜、镜头、字幕文件、图片或视频；用户另行要求时再路由相应 Skill。

## 其他平台

先读取用户给出的受众、格式、语气和限制，再采用“开头—展开—收束—CTA”的最小完整结构。若精确平台限制影响交付，优先查询官方当前规则，而不是类推其他平台。

# Style Guide

## Visual Identity

Use a hand-drawn whiteboard or notebook diagram style:

- Warm off-white paper background with subtle grain.
- Black or dark ink outlines, slightly wobbly and imperfect.
- Rounded boxes, casual arrows, simple icons, handwritten labels.
- Medium-low information density with generous blank space.
- Lightweight educational poster feeling, not a polished corporate slide.

Use `assets/reference-images/reference-01.jpeg` through `reference-04.jpeg` as style references when available. Borrow the paper feel, hand-drawn line quality, spacing, and annotation mood only. Do not reproduce their specific subjects, layout, or text.

Suggested background colors:

```text
#F6F1E7
#F8F3E8
#FAF7EE
```

Suggested line colors:

```text
#1F1F1F
#222222
#2A2A2A
```

## Typography

Use handwritten marker-style lettering. Keep labels short and large.

Rules:

- Use English by default.
- Use no more than 3-6 words per label when possible.
- Use one short title and optional short subtitle.
- Remove small footnotes and dense academic sentences.
- Make all text readable on mobile.

Good labels:

```text
The Request
What Came Back
Context Window
Verification Gap
Agent Writes File
Issue Found
PR Opens Clean
From AGI to ASI
Four Possible Pathways
Recursive Improvement
Flywheel or Brake?
```

Avoid formal serif fonts, corporate sans-serif fonts, cyberpunk fonts, cute bubble fonts, and dense typography.

## Palette

Use low-saturation pastel colors. Color should support meaning.

```text
Ink Black: #222222
Warm Paper: #F8F3E8
Soft Blue: #DCEBFA
Soft Green: #CDEFD7
Soft Red: #F4B6A8
Soft Orange: #F6C99A
Soft Yellow: #F7E6A6
Soft Purple: #DDD3F5
```

Semantic mapping:

- Blue: system, tool, dashboard, neutral info.
- Green: success, improvement, passed, enabled.
- Red: error, risk, failure, warning.
- Orange: cost, tokens, increase, attention.
- Yellow: highlight, key idea, discovery.
- Purple: AI, model, agent, abstract capability.

## Characters

Use simple cartoon people or agents only when they clarify the idea.

Human characters:

- Round head, minimal facial features, simple body.
- Clear expression: confused, curious, worried, thinking, surprised, relieved.
- Optional laptop, magnifying glass, notebook, or marker.

AI agent characters:

- Small friendly robot.
- Simple assistant figure with question marks.
- Chip with a face.
- Developer-agent hybrid figure.

Keep characters secondary to the explanation.

## Arrows and Charts

Use arrows to show causality:

- Straight arrows for direct flow.
- Curved arrows for loops.
- Dashed arrows for uncertainty or indirect relation.
- Circular arrows for retry, re-verification, or recursion.
- Branching arrows for pathways.

Keep crossings minimal.

Charts should be hand-drawn and simple: large value labels, minimal axes, pastel fills, and sketch outlines. Avoid detailed gridlines, tiny ticks, dashboards, 3D charts, and complex legends.

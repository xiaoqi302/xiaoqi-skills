# Composition Templates

## A. Human vs Agent

Use for explaining why humans and AI agents perceive systems differently.

Layout:

```text
Top: Human Developer
Right top: Full dashboard / system overview

Bottom: AI Agent
Right bottom: Scattered tool calls / missing information / fragmented state
```

Core message:

```text
Humans see the full system.
Agents discover it piece by piece.
```

## B. Request vs Returned Context

Use for tool over-fetching, context waste, and broad search results.

Layout:

```text
Left: User asks for one specific thing
Right: Huge response with unrelated modules
Bottom: Impact on context window
```

Core message:

```text
Every broad tool call returns more than the answer.
```

## C. Expected vs Actual

Use for surprising model behavior or metric differences.

Layout:

```text
Left: User expectation
Middle: Actual result chart
Right: Why this happened
```

Core message:

```text
A stronger model may explore more thoroughly.
```

## D. Without vs With Feedback Loop

Use for verification, testing, CI, code review, and agentic workflows.

Layout:

```text
Left: Without verification inside the loop
Right: With verification inside the loop
Bottom: One strong takeaway sentence
```

Core message:

```text
Verification belongs inside the loop, not after it.
```

## E. Four Possible Pathways

Use for papers, strategies, or technology roadmaps with multiple routes.

Layout:

```text
Left: Starting point
Middle: Four stacked pathway boxes
Right: Destination / goal
Bottom: Parallel-pathway takeaway
```

Example labels:

```text
From AGI to ASI
Four Possible Pathways

Scaling
Compute, models, and data keep growing

Paradigm Shift
New architectures, new algorithms

Recursive Improvement
AI improves AI

Group Agents
Many AGIs form a super-organization

These pathways may happen in parallel
```

## F. Concept Ladder

Use for comparing levels, definitions, or hierarchy.

Layout:

```text
Left low step: Basic concept
Middle higher step: Advanced concept
Right highest step: Theoretical limit
Bottom: Simple inequality or hierarchy
```

Example labels:

```text
AGI vs ASI vs UAI

AGI
One human-level intelligence

ASI
Beyond expert collectives

UAI / AIXI
Theoretical upper bound
Not computable

AGI < ASI < UAI
```

## G. Recursive Flywheel

Use for recursive improvement, feedback loops, compounding effects, and self-improvement.

Layout:

```text
Center: Circular loop
Left: Acceleration note
Right: Bottleneck list
Bottom: Question or takeaway
```

Example labels:

```text
Recursive Improvement
AI improves AI

AI Research
Better Model
Faster R&D
Even Better AI

May accelerate

Bottlenecks
Data
Chips
Energy
Experiments

Flywheel or brake?
```

## H. Single Agent vs Many Agents

Use for collective intelligence, multi-agent systems, organizations, and swarms.

Layout:

```text
Left: One agent
Right: Network of many agents
Far right: Outputs
Bottom: Strong takeaway
```

Example labels:

```text
Single AGI vs Many AGIs
ASI may not be a lone genius

Single AGI
One smart individual

Many AGIs
Super-organizational capability

Research Breakthroughs
Powerful Tools
High-Quality Products

ASI may be more like a company
```

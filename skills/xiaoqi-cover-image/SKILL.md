# Xiaoqi Cover Image

为内容做封面，不做花哨海报。封面要在 2 秒内传达主题、冲突和气质。

## 参考资源

- `references/cover-style.md`：平台比例、安全区和视觉检查。
- `references/prompt-template.md`：提示词模板。

## 两种模式

### 方向模式

用户只要方向、方案、灵感或提示词时：

1. 提炼主题、核心判断、读者情绪和主视觉隐喻。
2. 输出 3 个真正不同的方向：
   - `概念隐喻型`
   - `人物观察型`
   - `结构对比型`
3. 每个方向输出：
   - 标题建议
   - 画面构图
   - 文字策略
   - 图像生成提示词
4. 明确推荐一个方向并说明取舍。方向模式不调用生图工具。

### 直接生成模式

用户要求做图、生成或直接出图时：

1. 确认平台、比例和必须原样保留的标题；信息足够时不要追问。
2. 选最稳的一个方向，按 `references/prompt-template.md` 写单张提示词。
3. 调用当前环境可用的图像生成工具生成一张封面，不把 3 个方向拼成一张图。
4. 使用视觉查看工具检查标题、安全区、裁剪和缩略图表现。
5. 不通过时优先局部编辑；最多修复 2 轮，仍失败则交付最稳版本并简述限制。

## 平台默认值

- 公众号横封面：`2.35:1`。
- 通用文章头图、视频封面：`16:9`。
- 播客封面：`1:1`。
- 用户指定比例时，以用户要求为准。
- 用户提供标题时逐字保留，不擅自润色；未提供时标题控制在 8-14 个汉字。

## 画面原则

- 封面第一眼是视觉，不是标题。
- 中文标题最多 8-14 字。
- 留白要足，主体要大。
- 小七风格优先：黑白手绘、少量红蓝批注、清醒、克制、有判断。
- 不要科技蓝渐变背景，不要霓虹城市，不要抽象光效堆叠。
- 重要标题、人物面部和主物件必须位于平台安全区内。
- 缩小到手机列表缩略图后，标题和主视觉仍应一眼可辨。

## 输出格式

```markdown
## 方向 A

标题：
构图：
文字：
提示词：

## 方向 B
...

## 推荐

...
```

直接生成模式只交付最终封面、用途和保存位置；不要同时交付三张近似候选。

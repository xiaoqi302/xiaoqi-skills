# 小七公众号写作规则

## 结构

结构服务中心问题，不使用固定段数模板。

1. 开头从事件、矛盾、读者场景或关键变化中选择最自然的入口。
2. 在前部交代发生了什么，以及文章准备回答什么问题。
3. 正文按“事实或现象 → 人话解释 → 为什么重要”推进；一个小节只承担一个判断。
4. 结尾回到证据能支持的结论、限制或行动建议，不强行升华和互动。

材料不适合完整长文时，宁可缩短文章或交付结构，也不要用常识性空话填满篇幅。

## 事实边界

- 事实：材料明确提供，或已由可靠来源核实。
- 来源方主张：使用“据该公司称”“报告声称”等归属表达，不改写成客观结论。
- 作者判断：说明依据和边界，可使用“我的判断是”，但不伪造亲身体验。
- 待确认信息：保留为缺口或删除，不用模糊措辞掩盖。

材料缺少日期时，不补年份或“最近”；缺少数字、案例或引用时，不生成近似值和虚构例子。只能补文章结构、人话解释和不引入新事实的衔接。

## 语言

- 中文为主，仅保留有助于准确理解的英文术语。
- 使用自然长短句和短段落，标题表达问题或判断。
- 技术词首次出现时，用一个具体动作、结果或场景解释。
- 数据后解释其意义，同时说明口径或比较基准。
- 不默认使用 emoji、感叹号或营销形容词。

## 反模板规则

- 不固定用年份或“时代正在变化”开头。
- 不连续使用“真正重要的是”“这意味着什么”“值得注意的是”等路标句。
- 不为追求节奏强行写成三点、排比句或对称标题。
- 不机械安排“先泼冷水、再下判断、最后升华”。
- 不固定设置“说点自己的看法”章节；只有内容需要时才使用第一人称。
- 不在结尾例行加入关注、转发、评论或配图建议。

## 交付检查

- 文章的中心判断能否由现有事实支撑？
- 事实、来源方主张和作者判断是否能被读者区分？
- 每个标题是否推进问题，而不是只换一种说法？
- 是否出现了无来源的日期、数字、引语、案例或体验？
- 删除任何一段后若不影响信息量，该段是否应该直接删掉？

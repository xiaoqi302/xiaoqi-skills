# xiaoqi-wechat

把用户输入的文章、素材或提纲，改写成“小七公众号”风格：技术信息讲人话，趋势判断要明确，结构清楚，有数据、有案例、有自己的看法。成稿后按需建议使用 `$xiaoqi-cover-image` 生成封面，使用 `$xiaoqi-article-illustrator` 生成正文配图。

## 适用场景

使用本 skill：
- 用户要求“转成我的公众号风格”“小七风格”“xiaoqi 风格”。
- 用户给 AI 产品、模型发布、Agent、编程工具、开源项目、行业趋势类素材，要求改写成公众号文章。
- 用户给英文/技术报告/新闻稿，要求写成中文技术评论文章。

不使用本 skill：
- 用户只要新闻摘要、纯翻译、学术论文、广告软文、严肃白皮书。
- 用户要求保留原文结构逐句润色。

## 核心气质

小七风格的底层公式：

```text
技术事实 + 人话解释 + 趋势判断 + 一点个人体感
```

写作状态像一个懂技术、常用 AI 工具、愿意下判断的朋友在解释一件新变化。不要装腔，不要堆术语，不要营销腔。

## 文章结构

优先使用这个骨架：

1. **开头 3-6 段**
   - 用年份、行业变化、反差或异常现象开场。
   - 快速点出核心事件。
   - 给一个清晰判断：这不是普通更新，而是某种趋势变化。
   - 用一句“但要理解这件事，得先看清游戏规则怎么变了”之类的句子转入正文。

2. **正文 4-8 个二级标题**
   - 每节只讲一个观点。
   - 顺序通常是：背景变化 -> 核心能力 -> 数据/案例 -> 为什么重要 -> 未来影响。
   - 每节里穿插短句判断，避免变成资料堆砌。

3. **数据和案例**
   - 有数据就明确列出，使用项目符号或表格。
   - 数据后必须解释“这意味着什么”，不要只罗列。
   - 案例要服务观点，不要写成新闻通稿。

4. **结尾**
   - 用 `## 说点自己的看法` 或 `## 最后` 收束。
   - 先给最核心判断，再写 2-4 段个人分析。
   - 最后一段落到趋势、行动建议或评论互动。

## 常用句式

可以自然使用这些句式，但不要机械堆叠：

- “这不是……的故事，而是……”
- “真正有意思的是……”
- “理解这件事的关键，是……”
- “数据能说明一部分问题。”
- “这意味着什么？”
- “这不是后续会支持，而是发布即能用。”
- “我觉得更值得关注的不是……而是……”
- “热闹背后，也要泼一点冷水。”
- “这一步是险棋，也是妙棋。”
- “谁先把……做顺，谁就能拉开差距。”

## 语言规则

- 中文为主，关键技术词保留英文：Agent、SOTA、MCP、tokens、Workflow、Context Window。
- 段落短，通常 1-3 句一段。
- 多用判断句，少用长从句。
- 解释术语时用括号或一句人话说明，不写教材。
- 可以适度加粗关键判断，但不要整篇都是粗体。
- 标题直接、有判断，不要标题党。
- 不使用 emoji，除非用户明确要求。
- 不编造数据、链接、公司案例。素材里没有的数据，写“如果按公开信息看”或直接省略。

## 改写流程

1. **先提炼素材**
   - 事件是什么？
   - 为什么现在重要？
   - 核心变化是什么？
   - 有哪些数据、案例、争议？
   - 读者看完应该得到什么判断？

2. **重排结构**
   - 不照搬原文顺序。
   - 把最能代表趋势变化的点提前。
   - 把技术细节放在“为什么重要”的后面解释。

3. **写开头**
   - 第一段给时间或场景。
   - 第二段给事件。
   - 第三段给判断。
   - 第四段用转折进入正文。

4. **写正文**
   - 每节标题代表一个判断。
   - 先讲结论，再解释原因。
   - 遇到复杂技术，用“简单说”拆成 2-3 点。

5. **写个人看法**
   - 明确说“我觉得”“我的体感是”“更值得关注的是”。
   - 可以承认限制：不是万能药、仍然有门槛、还需要验证。
   - 最后落到 2026、下一轮竞争、开发者/团队该怎么做。

6. **推荐配图**
   - 文章定稿后，如果用户需要封面，使用 `$xiaoqi-cover-image`。
   - 如果用户需要正文配图或 shot list，使用 `$xiaoqi-article-illustrator`。
   - 默认建议 1 张封面 + 5-8 张正文配图；文章很短时可以少于 5 张，文章很长也不要轻易超过 8 张。
   - 配图要对应文章里的关键判断、结构转折、数据含义和最后观点，不要平均每节硬配一张。
   - 如果用户明确指定图片数量、比例或是否只要 shot list，按用户要求调整。

## 输出要求

默认输出完整公众号文章，不解释改写过程。文章之后用一句话给出建议的下一步配图 skill，例如“下一步可用 `$xiaoqi-cover-image` 做封面，用 `$xiaoqi-article-illustrator` 做 5-8 张正文配图。”

如果用户只给零散素材，先补成完整文章；如果信息明显不足，只问一个必要问题。不要把“缺少信息”当成停止理由。

输出前自检：
- 开头是否在 6 段内给出核心判断？
- 每个二级标题是否都在推进观点？
- 是否有“数据/案例 + 解释意义”？
- 是否有个人判断，而不是纯复述？
- 是否避免了编造事实？
- 结尾是否有清晰收束？
- 是否给出了合适的配图下一步，而不是绑定外部 skill？

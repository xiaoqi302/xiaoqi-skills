# 精确修复手册

## 指令结构

每条局部修复指令都要包含：

1. **区域**：唯一可定位的位置或元素。
2. **保留**：明确哪些像素、布局、角色或文字不能变化。
3. **改动**：替换、删除、移动、缩放或重画的准确动作。
4. **目标**：准确文字、尺寸、颜色、方向或风格结果。
5. **禁止**：不得新增的文字、图标、装饰、Logo 或构图变化。

示例：`只编辑标题下方第二个圆角框；保留其位置、尺寸和黑色边框，将乱码替换为“失败恢复”，使用与第一框相同字号和字重；不要改动其他框、箭头或背景。`

## 按问题修复

- **比例或裁剪**：扩展画布或缩小主体，锁定人物与文字比例；不要直接拉伸整图。明确目标比例和安全边距。
- **文字错误**：只编辑目标文字区域，提供必须逐字一致的替换文本，并锁定相邻图形。大面积乱码优先重生成无字底图，再可靠叠字。
- **层级混乱**：缩小或降对比次要元素，放大核心主体；不要同时重做全部元素。
- **过密或留白不足**：删除明确列出的次要元素，扩大指定边距；不要用缩小全部文字换取空间。
- **风格漂移**：引用已查看参考图的具体线条、背景和配色特征，只重画漂移区域；整体漂移直接重生成。
- **技术关系错误**：锁定节点文字，纠正指定箭头方向、连接端点、公式或数值；修后逐条复核关系。
- **IP 角色漂移**：保留动作与场景，按参考图修正脸型、身体比例、服装轮廓和配色；已像其他 IP 时重生成角色区域或整图。
- **伪 Logo、水印或随机角标**：删除指定区域并用邻近背景自然补全；不得替换成另一个虚构标识。

## 两轮上限

### 第 1 轮

- 先修全部 P0，再处理阻塞理解的 P1。
- 尽量合并互不重叠的局部修改，保留未出问题区域。
- 查看新生成的真实图片，重新记录证据和结论。

### 第 2 轮

- 只处理复检后仍存在的 P0/P1，或第一轮引入的新问题。
- 再次查看真实图片并完整复检。
- 两轮后仍有 P0，或局部编辑持续破坏其他区域，停止修图并判定 `REGENERATE`。

## 动作边界

- 只评审时，不调用编辑工具。
- 明确要求修图但无工具时，输出可复制的精确指令，并写明“未实际修改”。
- 每次编辑另存新文件，保留原图；只有用户明确要求时才覆盖。
- 未查看修复结果时，不得写“已修复”“已通过”或类似结论。

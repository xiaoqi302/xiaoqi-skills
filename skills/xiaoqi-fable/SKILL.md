# 小七寓言配图

执行已经确定的画面方案：把一个核心判断变成一张安静、克制的未来寓言图，或按已确认 shot list 逐张生成系列图。默认使用小白 IP、16:9 横版、黑白手绘、少量红蓝批注和大量留白。

## 边界

- 已指定段落、观点、隐喻、场景或已确认 shot list：由本 Skill 生成或编辑。
- 只有整篇文章，尚未决定哪里配图、画几张或用什么画面：先用 `$xiaoqi-article-illustrator` 做选点与路由；本 Skill 不自行输出整篇 shot list。
- 每张图只表达一个核心结构，不把多段文章压进一张图。

## 按需读取

- [references/style-dna.md](references/style-dna.md)：风格、颜色和文字密度。
- [references/xiaobai-ip.md](references/xiaobai-ip.md)：角色形象、动作与系列一致性。
- [references/composition-patterns.md](references/composition-patterns.md)：原创隐喻与反复刻规则。
- [references/prompt-template.md](references/prompt-template.md)：单张提示词模板。
- [references/qa-checklist.md](references/qa-checklist.md)：真实成图检查与修复顺序。
- [examples/prompts.md](examples/prompts.md)：只校准提示词密度，不照抄构图。

## 工作流

### 1. 规范化已确定画面

为每个 shot 明确：核心意思、物理隐喻、小白动作、3-6 个关键物件和 `3-6` 个短中文批注。用户只指定某一段或某个观点时，可以为该选点推导一个 shot；不要借此重新规划整篇文章。

### 2. 锁定小白一致性

- 用户提供参考图时，先查看参考图，再把它传给支持参考图的生成或编辑工具；不要描述自己未查看的参考图。
- 没有参考图时，使用 [references/xiaobai-ip.md](references/xiaobai-ip.md) 写一段固定“角色锚点”，并在系列每张提示词中逐字复用角色轮廓、脸、外套、四肢比例和配色描述。
- 工具支持固定种子或图像参考时，复用同一种子，或把首张通过质检的成图作为后续参考；工具不支持时，只依靠同一角色锚点，不声称已有参考资产。

### 3. 生成或提示词交付

- 用户要求“生成 / 做图 / 出图”且环境有图像生成或编辑工具时，使用当前可用工具实际生成，不绑定具体供应商。
- 用户明确只要提示词，或环境没有图像工具时，按模板输出逐张完整提示词并标注“仅提示词”；不要声称已生成图片。
- 默认比例为 `16:9`，用户指定比例时严格跟随。每个 shot 单独生成，不把系列拼成一张。
- 每张只保留 `3-6` 个短中文批注；红色标风险或转折，蓝色标状态或第二层旁注。

### 4. 查看并修复

查看每张真实成图并按 [references/qa-checklist.md](references/qa-checklist.md) 检查。优先局部编辑，无法修复时重生成；每张图最多修复 `2` 轮。两轮后仍有问题，交付最优版本并明确剩余缺陷，不无限重试。

### 5. 安全落盘

1. 用户给出绝对输出目录时，使用该目录；用户给出相对目录时，基于绝对工作区根解析，并确认结果仍在工作区内。
2. 未指定目录时，先解析绝对工作区根，再保存到 `<绝对工作区根>/assets/<article-slug>-fable/`。
3. 如果无法确定工作区，保留工具返回的原始绝对路径，不复制到含义不明的相对目录。
4. 文件按 `01-topic-name.png`、`02-topic-name.png` 顺序命名；已有同名文件时添加新后缀，不覆盖。保留原始生成文件，修复版另存。

## 交付

- 实际生成：列出每张图的用途、绝对文件路径、检查结果和剩余问题。
- 仅提示词：按 shot 顺序给出完整提示词，不虚构文件路径。
- 输出保持简短，不复述整套风格理论。

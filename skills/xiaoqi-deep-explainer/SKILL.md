# Xiaoqi Deep Explainer

把复杂东西讲清楚，不把它讲浅。核心目标是让读者知道：它是什么、为什么现在重要、机制怎么工作、边界在哪里。

## 参考资源

- `references/explainer-structure.md`：解释结构。
- `references/analogy-rules.md`：比喻和例子规则。

## 工作流

1. 先判断读者层级：默认面向懂一点技术、但不想读论文细节的人。
2. 提炼一个中心问题：这件事到底解决了什么麻烦？
3. 分 5 层解释：
   - 一句话定义。
   - 背景问题。
   - 核心机制。
   - 一个具体例子。
   - 局限和误解。
4. 如果用户给论文或技术文档，保留关键术语和必要公式含义，但不要堆推导。
5. 结尾给“可以继续变成什么”：公众号文章、信息卡、流程图或短快评。

## 输出格式

```markdown
## 一句话

## 它解决的问题

## 核心机制

## 举个例子

## 容易误解的地方

## 可以怎么继续用
```

## 禁忌

- 不用“简单来说”掩盖关键细节。
- 不编论文实验结果。
- 不把类比当机制本身。
- 不做长篇百科式背景铺陈。

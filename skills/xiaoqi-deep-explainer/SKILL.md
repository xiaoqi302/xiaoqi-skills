# Xiaoqi Deep Explainer

把复杂内容讲清楚，但不通过删掉关键机制来换取“通俗”。

## 参考资源

- 读取 [深度解释结构](references/explainer-structure.md)，选择解释深度并区分主张层级。
- 需要类比时读取 [类比规则](references/analogy-rules.md)。

## 深度选择

- 快速理解：一句话定义、核心问题、关键机制和一个边界。
- 标准解释：默认层级，覆盖背景、机制、例子、差异与局限。
- 技术深挖：用户明确要求时，补充假设、流程、公式含义、证据和失败条件。

若用户没有指定，按其背景和问题选择最低但足够回答的层级；不要自动写成百科全书。

## 工作流

1. 明确读者背景、中心问题和所需深度。
2. 提取术语、机制链、证据与限制。
3. 用户提供论文或文档时，分开处理：
   - 原文主张：材料明确声称或实验直接报告的内容。
   - 解释：为了理解而做的准确转述。
   - 推断：由材料延伸出的分析，必须显式标注。
4. 按“问题 → 机制 → 具体例子 → 与旧方法的差异 → 边界”递进。
5. 核对例子和类比是否改变了机制，核对推断是否被误写成原文结论。

## 输出

结构随深度调整。涉及来源材料时，在关键位置使用“原文主张”“解释”“我的推断”区分层级；没有推断时不要为凑结构硬加一段。

## 底线

- 不编造论文结论、实验结果、公式含义或产品能力。
- 不把类比当机制，也不让类比覆盖关键限制。
- 不用“简单来说”跳过真正困难的环节。
- 不增加与中心问题无关的背景史。

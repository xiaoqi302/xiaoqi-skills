# 正文配图拆解策略

优先给这些位置配图：

1. 开头核心判断。
2. 旧规则和新规则的切换。
3. 技术机制第一次出现。
4. 数据背后的含义。
5. 风险、误解或冷水。
6. 结尾的行动判断。

不要给纯过渡段、背景资料堆叠段、引用段硬配图。

## 段落锚点

每个 shot 必须记录：

- 段落编号或小标题。
- 该段开头 12-30 字原文。
- 建议放在该段之前还是之后。

无法在文章中找到原文锚点时，不保留该 shot。

## 风格路由

| 内容功能 | 执行 Skill | 判断信号 |
|---|---|---|
| 寓言式观点图 | `$xiaoqi-fable` | 趋势、风险、选择、时代变化、认知转折 |
| 技术信息卡 | `$xiaoqi-info-card` | 架构、机制、步骤、数据关系、算法流程 |
| 极简解释卡 | `$xiaoqi-white` | 单概念、清单、公式、简短前后对比 |

优先选择最能解释该段的 Skill，不为了视觉统一强行全部使用小白寓言风。
